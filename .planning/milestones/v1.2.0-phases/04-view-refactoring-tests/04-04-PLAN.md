Refactor MapContentView in MapView.swift to be a declarative shell:

1. Replace all extracted @State properties with @StateObject:
```swift
@StateObject private var viewModel: MapViewModel
```

2. Add init:
```swift
init(robot: RobotConfig, robotManager: RobotManager, isFullscreen: Bool = false) {
    self.robot = robot
    _viewModel = StateObject(wrappedValue: MapViewModel(robot: robot, robotManager: robotManager, isFullscreen: isFullscreen))
}
```

3. Keep these @State in the View (gesture/drawing state is inherently view-local):
   - @State for pan/zoom gesture tracking (currentScale, currentOffset, etc.)
   - @GestureState properties
   - @State for touch/drag coordinates during zone drawing
   - @State for Canvas rendering parameters

4. Replace data reads: `map` -> `viewModel.map`, `editMode` -> `viewModel.editMode`, `restrictions` -> `viewModel.restrictions`

5. Replace action closures with viewModel calls:
   - Zone clean button -> await viewModel.cleanZones()
   - Go-to tap -> await viewModel.goToPoint(x:y:)
   - Room rename -> await viewModel.renameRoom(id:name:)
   - etc.

6. Replace .task { } with .task { await viewModel.loadMap(); viewModel.startMapRefresh() }
   Add .onDisappear { viewModel.stopMapRefresh() }

7. MapTabView wrapper: pass robotManager to MapContentView init.

8. MapPreviewView: This is a simple preview — it does NOT need the full MapViewModel. Leave it with its own lightweight @State (map, restrictions, isLoading). It only fetches map data for display, no editing.

9. Update any call sites that create MapContentView to pass robotManager.

Build and verify compilation.
