**1. UpdatePhase Enum** (per D-01 UpdatePhase Enum Design):
```swift
enum UpdatePhase: Equatable {
    case idle
    case checking
    case updateAvailable
    case downloading
    case readyToApply
    case applying
    case rebooting
    case error(String)

    static func == (lhs: UpdatePhase, rhs: UpdatePhase) -> Bool {
        switch (lhs, rhs) {
        case (.idle, .idle), (.checking, .checking),
             (.updateAvailable, .updateAvailable), (.downloading, .downloading),
             (.readyToApply, .readyToApply), (.applying, .applying),
             (.rebooting, .rebooting):
            return true
        case (.error(let a), .error(let b)):
            return a == b
        default:
            return false
        }
    }
}
```

**2. UpdateService Class** (per D-02 UpdateService Architektur):
- `@MainActor class UpdateService: ObservableObject`
- `@Published private(set) var phase: UpdatePhase = .idle`
- `private let logger = Logger(subsystem: Bundle.main.bundleIdentifier ?? "com.valetudio", category: "UpdateService")`
- `private let api: ValetudoAPI`
- `private var pollingTask: Task<Void, Never>?`
- `init(api: ValetudoAPI)`
- `deinit` mit `pollingTask?.cancel()`

**3. Methoden:**

`checkForUpdates()`:
- Guard: `guard case .idle = phase else { logger.warning(...); return }`
- `phase = .checking`
- `try await api.checkForUpdates()` dann `let state = try await api.getUpdaterState()`
- `phase = mapUpdaterState(state)`
- Catch: `phase = .error(error.localizedDescription)` (per D-03 Error-State)

`startDownload()`:
- Guard: `guard case .updateAvailable = phase else { logger.warning(...); return }` (per D-04 Re-Entrancy-Guard)
- `phase = .downloading`
- `try await api.downloadUpdate()`
- `await pollUntilReadyToApply()`
- Catch: `phase = .error(error.localizedDescription)`

`startApply()`:
- Guard: `guard case .readyToApply = phase else { logger.warning(...); return }` (per D-04)
- `phase = .applying`
- `try await api.applyUpdate()`
- `phase = .rebooting`
- Catch: `phase = .error(error.localizedDescription)`

`reset()`:
- `pollingTask?.cancel()`
- `phase = .idle`

**4. Private Hilfsmethoden:**

`pollUntilReadyToApply()`:
- Speichert `Task` in `pollingTask`
- Loop: max 60 Iterationen, `try? await Task.sleep(for: .seconds(5))`
- Guard `!Task.isCancelled`
- `try? await api.getUpdaterState()` — bei Fehler `continue`
- Mapped State pruefen: wenn `.readyToApply` → return
- Pitfall 6: Wenn State unerwartet `.idle` wird waehrend `.downloading` erwartet → `phase = .error("Download wurde unterbrochen")` und return
- Nach Loop-Ende ohne Ergebnis: `phase = .error("Download-Timeout")`
- `await pollingTask?.value` nach Task-Erstellung

`mapUpdaterState(_ state: UpdaterState) -> UpdatePhase`:
- `"ValetudoUpdaterIdleState"` → `.idle`
- `"ValetudoUpdaterApprovalPendingState"` → `.updateAvailable`
- `"ValetudoUpdaterDownloadingState"` → `.downloading`
- `"ValetudoUpdaterApplyPendingState"` → `.readyToApply`
- `default` → `logger.warning("Unknown updater state: ...")` → `.idle`
