## Spacing Scale

iOS spacing follows an 8-point grid. These values match existing `padding` usage in the codebase.

| Role | Value | Usage |
|------|-------|-------|
| xs | 4pt | Icon-to-text gap in HStack rows (frame(width: 24) icon + spacing) |
| sm | 8pt | Intra-section row spacing, VStack spacing in ControlButton |
| md | 16pt | List section inset (default SwiftUI List padding) |
| lg | 24pt | Between major sections in List |
| xl | 32pt | N/A for this phase |

Exceptions:
- Progress bar height: 8pt (fixed — reused from ValetudoInfoView memory bar pattern)
- ControlButton height: 56pt fixed (existing contract — do not change)
- Minimum touch target: 44pt (HIG requirement — met by List rows)

## Typography

All roles use SF Pro via SwiftUI text styles. Do not use fixed pixel sizes.

| Role | SwiftUI Style | Weight | Usage |
|------|---------------|--------|-------|
| Body | `.body` | regular (400) | LabeledContent labels and values, HStack row text |
| Caption | `.caption` | regular (400) | Commit hash (monospaced), secondary annotations |
| Caption2 | `.caption2` | regular (400) | ControlButton labels (existing contract) |
| Headline / Label header | `.headline` | semibold (600) | Section headers via `Label(_, systemImage:)` |

Special: Commit hash uses `.font(.system(.caption, design: .monospaced))` — reused from ValetudoInfoView line 846.

Weights in use: regular (400) and semibold (600) — no other weights.

## Color

All colors use SwiftUI semantic tokens. No hex values.

| Role | SwiftUI Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `.systemBackground` / `Color(uiColor: .systemBackground)` | List background |
| Secondary (30%) | `.secondarySystemGroupedBackground` | Grouped List section cells |
| Accent (10%) | `.accentColor` (tint) | Reserved for interactive SF Symbols only (see below) |
| Destructive | `.red` | Not used in this phase — no destructive actions |
| Secondary text | `.secondary` (foregroundStyle) | All read-only value labels in LabeledContent / HStack rows |

Accent color reserved for:
- SF Symbol icon in the DisclosureGroup header label
- Progress bar fill when memory usage is below 80% (`.blue`)
- Progress bar fill when memory usage is above 80% (`.red`) — this is a semantic warning, not destructive

Color rules for the memory/CPU progress bar (reused from ValetudoInfoView lines 922-931):
- Fill: `usedPercent > 0.8 ? Color.red : Color.blue`
- Track: `Color.secondary.opacity(0.2)`

## Component Inventory

These are the exact SwiftUI primitives and patterns to use. All are established in the codebase already.

### DeviceInfoSectionView (new Sub-View in RobotDetailSections.swift)

```
DisclosureGroup {
    // Hardware group
    LabeledContent("Model", value: props.model)
    LabeledContent("Hersteller", value: props.manufacturer)
    LabeledContent("Seriennummer", value: props.serial)

    // Valetudo group
    LabeledContent("Version", value: version.release)
    HStack { Text("Commit").foregroundStyle(.secondary); Spacer(); Text(commit.prefix(8)).font(.system(.caption, design: .monospaced)) }

    // System group
    LabeledContent("Hostname", value: hostInfo.hostname)
    LabeledContent("Uptime", value: formatUptime(hostInfo.uptime))

    // CPU bar
    ProgressView(value: cpuLoad, total: 1.0)  // or Gauge — see note
      .frame(height: 8)

    // Memory bar
    ZStack(alignment: .leading) {
        RoundedRectangle(cornerRadius: 4).fill(Color.secondary.opacity(0.2))
        RoundedRectangle(cornerRadius: 4).fill(usedPercent > 0.8 ? Color.red : Color.blue)
          .frame(width: geometry.size.width * usedPercent)
    }
    .frame(height: 8)

} label: {
    Label("Geräteinformationen", systemImage: "cpu")
}
```

CPU display note: ValetudoInfoView displays load as three float values (1m/5m/15m). CONTEXT.md requires a single bar. Use 1-minute load average normalized against an assumed maximum of `nCores` (or clamp to 1.0). If CPU core count is unavailable, use `load._1 / 4.0` as reasonable default and clamp to `min(value, 1.0)`.

### Loading state

Use existing pattern from ValetudoInfoView:
```swift
.overlay {
    if isLoading && version == nil {
        ProgressView()
    }
}
```

Applied at the Section level (not full-screen) when data is not yet loaded.

### DisclosureGroup — collapsed by default

```swift
@State private var isExpanded = false
DisclosureGroup(isExpanded: $isExpanded) { ... } label: { ... }
```

Default state: collapsed (`isExpanded = false`). Consistent with Statistics Section.

## Data Loading Contract

### ViewModel additions (RobotDetailViewModel)

New `@Published` properties to add:
```swift
@Published var valetudoVersion: ValetudoVersion?
@Published var systemHostInfo: SystemHostInfo?
@Published var isLoadingDeviceInfo = false
```

New method `loadDeviceInfo()` — mirrors `loadRobotProperties()` pattern:
```swift
func loadDeviceInfo() async {
    guard let api = robotManager.getAPI(for: robot.id) else { return }
    isLoadingDeviceInfo = true
    defer { isLoadingDeviceInfo = false }
    do {
        valetudoVersion = try await api.getValetudoVersion()
        systemHostInfo = try await api.getSystemHostInfo()
    } catch {
        logger.error("Failed to load device info: \(error, privacy: .public)")
    }
}
```

Load trigger: Called from `.task {}` in RobotDetailView alongside other existing load calls. No separate `.task` needed — group with `loadRobotProperties()`.
