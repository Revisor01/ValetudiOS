# Milestone v1.4.0: Code Quality & Robustness

**Status:** ✅ SHIPPED 2026-03-29
**Phases:** 9-11
**Total Plans:** 7

## Overview

Codebase sauber gemacht — alle print()-Reste eliminiert, Force-Unwraps entfernt, fehlende Logger ergänzt, inkonsistente Concurrency-Patterns migriert, Keychain-Fehlerbehandlung verbessert, und drei monolithische Views aufgebrochen.

## Phases

### Phase 9: Logger Migration
**Goal**: Kein einziger print()-Aufruf verbleibt in Views oder Services; alle Concurrency-Patterns folgen Swift Structured Concurrency
**Depends on**: Nothing (first phase of new milestone)
**Requirements**: LOG-01, LOG-02, LOG-03, SAFE-03
**Plans**: 3 plans

Plans:
- [x] 09-01-PLAN.md — Logger-Migration Views (Batch 1): DoNotDisturbView, StatisticsView, IntensityControlView, MapView (11 print()-Stellen)
- [x] 09-02-PLAN.md — Logger-Migration Views (Batch 2): ManualControlView, RoomsManagementView, TimersView (17 print()-Stellen)
- [x] 09-03-PLAN.md — Logger-Migration SupportManager + DispatchQueue→Task.sleep in SupportReminderView

### Phase 10: Safety Fixes
**Goal**: Kein Force-Unwrap gefährdet die App-Stabilität, Keychain-Fehler werden sichtbar geloggt, und alle Magic-Strings sind zentralisiert
**Depends on**: Phase 9
**Requirements**: SAFE-01, SAFE-02, ORG-01
**Plans**: 1 plan

Plans:
- [x] 10-01-PLAN.md — SAFE-01/02/ORG-01: Force-Unwrap fix, Keychain-Fehlerlogger, Constants.swift mit URLs und ProductIDs

### Phase 11: View Decomposition
**Goal**: Die drei größten Views sind in überschaubare Sub-Views aufgeteilt; keine einzelne View-Datei überschreitet eine handhabbare Größe
**Depends on**: Phase 10
**Requirements**: ORG-02, ORG-03, ORG-04
**Plans**: 3 plans

Plans:
- [x] 11-01-PLAN.md — MapView-Dekomposition: MiniMapView, InteractiveMapView, MapControlBarsView, MapSheetsView

- [x] 11-03-PLAN.md — RobotDetailView-Dekomposition: RobotDetailSections.swift mit ControlButton, DockActionButton, PulseAnimationView

## Milestone Summary

**Key Accomplishments:**
- 30 print() → os.Logger migration in 8 Views + SupportManager
- Force-Unwrap in SettingsView eliminiert (nil-coalescing)
- Keychain-Fehlerbehandlung mit OSStatus-Logging
- DispatchQueue → Task.sleep structured concurrency
- Constants.swift für zentralisierte URLs/ProductIDs
- MapView: 2532 → 859 Zeilen (66% Reduktion)
- RobotSettingsView: 1801 → 502 Zeilen (72% Reduktion)
- RobotDetailView: 1253 → 1125 Zeilen + Helper-Extraktion

**Key Decisions:**
- Sub-Structs in Views (SplitSegmentSheet, TimerEditView) erhielten eigene Logger-Properties
- MapContentView Control Bars als Swift Extension (nicht eigenständige Structs) — State-Zugriff erfordert dies
- KeychainStore nutzt static Logger (struct, nicht class)
- Constants als enum (nicht struct) — verhindert versehentliche Instanziierung

**Issues Resolved:**
- Alle 30 print()-Aufrufe aus Views eliminiert
- Force-Unwrap Crash-Risiko behoben
- 9 duplizierte ProductID-Strings zentralisiert
- 2 silent Keychain-Fehler jetzt sichtbar

**Technical Debt Incurred:**
- None — this milestone was designed to eliminate tech debt
