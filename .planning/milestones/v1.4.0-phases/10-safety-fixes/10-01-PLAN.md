    KeychainStore.swift hat bereits einen Logger aus Phase 1 (`private let logger = Logger(...)`). Falls nicht vorhanden, nach dem Muster hinzufügen:
    ```swift
    private let logger = Logger(subsystem: Bundle.main.bundleIdentifier ?? "ValetudiOS", category: "KeychainStore")
    ```

    Zeile 28 (erster SecItemDelete, beim Speichern): Return-Wert prüfen und loggen:
    ```swift
    // VORHER:
    SecItemDelete(deleteQuery as CFDictionary) // errSecItemNotFound is OK

    // NACHHER:
    let deleteStatus = SecItemDelete(deleteQuery as CFDictionary)
    if deleteStatus != errSecSuccess && deleteStatus != errSecItemNotFound {
        logger.error("SecItemDelete (save) failed: \(deleteStatus, privacy: .public)")
    }
    ```

    Zeile 46 (zweiter SecItemDelete, beim Löschen): Return-Wert prüfen und loggen:
    ```swift
    // VORHER:
    SecItemDelete(query as CFDictionary)

    // NACHHER:
    let deleteStatus = SecItemDelete(query as CFDictionary)
    if deleteStatus != errSecSuccess && deleteStatus != errSecItemNotFound {
        logger.error("SecItemDelete (delete) failed: \(deleteStatus, privacy: .public)")
    }
    ```

    SecItemAdd ist bereits durch `== errSecSuccess` geprüft — falls es einen return-Wert ignoriert, ebenfalls loggen. Den genauen Kontext aus der Datei lesen und nur die beiden SecItemDelete-Stellen anpassen.
  </action>

    - `grep "logger.error" KeychainStore.swift` findet mindestens 2 Treffer
