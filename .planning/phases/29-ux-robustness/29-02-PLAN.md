    2. Neue private Methode `restartRefreshing()` hinzufuegen:

       }

    3. In `startRefreshing()` den HTTP-Polling-Teil (die `withTaskGroup` Sektion, Zeilen ~161-170) aendern:
       - SSE-Connect-Loop (Zeilen ~138-157) bleibt UNVERAENDERT — SSE fuer alle Roboter.
       - HTTP-Polling TaskGroup: Wenn `activeRobotId != nil`, nur den aktiven Roboter pollen. Wenn `activeRobotId == nil` (kein Roboter ausgewaehlt), alle wie bisher pollen.

       Neuer Code fuer die TaskGroup:
       ```swift
       // Poll only robots without active SSE (fallback)
       // When activeRobotId is set, only poll the active robot
       let robotsToPoll = activeRobotId != nil
           ? robots.filter { $0.id == activeRobotId }
           : robots

       await withTaskGroup(of: Void.self) { group in
           for robot in robotsToPoll {
               group.addTask {
                   let sseActive = await self.sseManager.isSSEActive(for: robot.id)
                   if !sseActive {
                       await self.refreshRobot(robot.id)
                   }
               }
           }
       }
       ```

    WICHTIG (Research Pitfall 3): SSE-Verbindungen fuer ALLE Roboter beibehalten. Nur das HTTP-Fallback-Polling wird beschraenkt. Sonst fehlen Batterie-/Status-Updates fuer nicht-aktive Roboter.
