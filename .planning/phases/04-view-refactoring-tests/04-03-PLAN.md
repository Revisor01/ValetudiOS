Refactor RobotSettingsView to be a declarative shell:

1. Replace all extracted @State properties with a single @StateObject:
```swift
@StateObject private var viewModel: RobotSettingsViewModel
```

2. Add init:
```swift
init(robot: RobotConfig, robotManager: RobotManager) {
    self.robot = robot
    _viewModel = StateObject(wrappedValue: RobotSettingsViewModel(robot: robot, robotManager: robotManager))
}
```

3. Replace all `$propertyName` bindings with `$viewModel.propertyName` for two-way bindings (e.g., Slider value, Toggle isOn).

4. Replace all inline Task { } action closures with viewModel method calls.

5. Replace .task { ... loading block } with .task { await viewModel.loadSettings() }

6. Keep ONLY these @State in the View (pure UI presentation):
   - @State private var showMappingAlert: Bool (alert presentation)
   - @State private var showMapResetAlert: Bool (alert presentation)
   Any other alert/sheet toggles that are purely UI.

7. All `if hasVolumeControl`, `if hasCarpetMode` etc. become `if viewModel.hasVolumeControl`, etc.

8. Keep @EnvironmentObject var errorRouter: ErrorRouter in View.

9. Update any call sites that create RobotSettingsView to pass robotManager if needed. Check SettingsView.swift and any .sheet presentations.
