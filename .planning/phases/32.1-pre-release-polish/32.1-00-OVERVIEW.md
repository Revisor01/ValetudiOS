# Phase 32.1: Pre-Release Polish

## Goal

Letzte Pre-Submission-Fixes: Bonjour-Discovery liefert korrekte Hostnamen + IPs, Settings-About-Sektion ist aufgeräumt.

## Context

Während App-Store-Submission-Vorbereitung wurden zwei Bugs/Issues entdeckt:

1. **Bonjour-Discovery-Bug**: Automatischer Netzwerkscan fand zwar Valetudo-Roboter via Bonjour, baute aber den Hostnamen aus dem Service-Instanznamen (`Valetudo CriticalMetallicMole`) statt aus dem TXT-Record-Feld `id` — Resultat war ein unbrauchbarer Host mit Leerzeichen. Zusätzlich fand der IP-Scan falsche Geräte (`.36`/`.38` statt `.35`/`.40`), weil nur HTTP 200 als Valetudo-Indikator akzeptiert wurde — Roboter mit aktivem Basic Auth (HTTP 401) wurden ignoriert, während ein Fremdgerät mit zufälligem 200 OK fälschlicherweise erkannt wurde.

2. **About-Section UI**: Version war doppelt (Zeile in Sektion + im Footer-Claim "ValetudiOS v1.0.0"). Aufgeräumt: nur Footer behalten, Sektion zeigt nur noch Valetudo Website + Valetudo GitHub + ValetudiOS GitHub.

## Changes

### Bonjour Discovery Bugfix
- `ValetudoApp/Services/NWBrowserService.swift`: TXT-Record-Feld `id` (lowercased) für Hostname (`valetudo-<id>.local`); TXT-Key `name` mit Fallback auf `friendlyName`; defensiver Sanitize-Fallback.
- `ValetudoApp/Services/NetworkScanner.swift`: Identifikation per `X-Valetudo-Version` Response-Header (`value(forHTTPHeaderField:)` ist case-insensitive); akzeptiert HTTP 200 oder 401.

### About Section Cleanup
- `ValetudoApp/Views/SettingsView.swift`: Version-Zeile (`settings.version` + `CFBundleShortVersionString`) aus About-Sektion entfernt — Version steht weiterhin im Footer-Claim.

## Verification

- `xcodebuild -project ValetudoApp/ValetudoApp.xcodeproj -scheme ValetudoApp -sdk iphonesimulator -configuration Debug build` → `** BUILD SUCCEEDED **`
- Debug-Session-Artefakt: `.planning/debug/resolved/bonjour-discovery-wrong-host-ip.md`

## Files Changed

- `ValetudoApp/ValetudoApp/Services/NWBrowserService.swift`
- `ValetudoApp/ValetudoApp/Services/NetworkScanner.swift`
- `ValetudoApp/ValetudoApp/Views/SettingsView.swift`

## Real-Device-Test (empfohlen vor Submission)

Im selben WLAN App auf echtem Gerät starten → Scan ausführen → es sollten zwei Bonjour-Einträge `valetudo-criticalmetallicmole.local` und `valetudo-digitalslowechidna.local` erscheinen, mit korrekten IPs `.35`/`.40`. Fremdgerät auf `.36` darf nicht mehr auftauchen.
