  <action>
    **1. Import hinzufuegen:**
    `import BackgroundTasks` am Anfang der Datei (neben import SwiftUI und import UserNotifications).

    **2. AppDelegate.didFinishLaunchingWithOptions erweitern:**
    NACH `UNUserNotificationCenter.current().delegate = self` und VOR `return true`:

    ```swift
    // BGTask-Handler registrieren — MUSS in didFinishLaunchingWithOptions passieren
    BGTaskScheduler.shared.register(
        forTaskWithIdentifier: BackgroundMonitorService.taskIdentifier,
        using: nil
    ) { task in
        BackgroundMonitorService.shared.handleBackgroundRefresh(task: task as! BGAppRefreshTask)
    }

    // Initiales Scheduling (fuer frische Installationen die noch nie in den Hintergrund gingen)
    BackgroundMonitorService.shared.scheduleBackgroundRefresh()
    ```

    Hinweis: `task as! BGAppRefreshTask` ist hier sicher, da der registrierte Identifier nur fuer BGAppRefreshTask verwendet wird.

    **3. ValetudoApp struct erweitern — scenePhase fuer Hintergrund-Scheduling:**
    In der `ValetudoApp` struct:
    a. `@Environment(\.scenePhase) private var scenePhase` Property hinzufuegen
    b. `.onChange(of: scenePhase)` Modifier an die WindowGroup anfuegen (nach den bestehenden .environmentObject und .onAppear Modifiern):

    ```swift
    .onChange(of: scenePhase) { _, newPhase in
        if newPhase == .background {
            BackgroundMonitorService.shared.scheduleBackgroundRefresh()
        }
    }
    ```

    Der onChange-Modifier soll an BEIDE Branches (hasCompletedOnboarding true/false) angefuegt werden, oder besser: den gesamten if/else-Block in eine Group wrappen und onChange darauf.

    Einfachster Ansatz: onChange an die WindowGroup selbst haengen (nicht an die inneren Views):
    ```swift
    WindowGroup {
        // ... bestehender if/else Block
    }
    .onChange(of: scenePhase) { _, newPhase in
        if newPhase == .background {
            BackgroundMonitorService.shared.scheduleBackgroundRefresh()
        }
    }
    ```

    KEIN `applicationDidEnterBackground` in AppDelegate noetig — SwiftUI Scene-Lifecycle nutzt scenePhase stattdessen. AppDelegate's applicationDidEnterBackground wird bei SwiftUI App-Lifecycle nicht aufgerufen.
