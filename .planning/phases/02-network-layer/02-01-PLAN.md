```swift
private lazy var sseSession: URLSession = {
    let config = URLSessionConfiguration.default
    config.timeoutIntervalForRequest = .infinity
    config.timeoutIntervalForResource = .infinity
    // SSL delegate analog zu bestehender session wenn config_useSSL_and_ignoreCertErrors
    if let delegate = sessionDelegate {
        return URLSession(configuration: config, delegate: delegate, delegateQueue: nil)
    }
    return URLSession(configuration: config)
}()
```

Zwei neue Methoden:
- `func streamStateLines() async throws -> URLSession.AsyncBytes` — GET `/api/v2/robot/state/attributes/sse`, Basic Auth Header analog zu bestehender `request()` Methode, nutzt `sseSession`
- `func streamMapLines() async throws -> URLSession.AsyncBytes` — GET `/api/v2/robot/state/map/sse`, gleiche Authentifizierung, nutzt `sseSession`

Beide Methoden: URL aus `config.baseURL` + Pfad bauen, Auth-Header setzen (Muster aus bestehender `request()` Methode kopieren — dort wird `config.username` + `KeychainStore.password(for:)` genutzt), `sseSession.bytes(for: request)` aufrufen, HTTP 200 pruefen, `bytes` zurueckgeben.

**2. SSEConnectionManager actor (per D-02, D-03, D-04):**

Neue Datei `SSEConnectionManager.swift` erstellen:
- Swift `actor` (NICHT ObservableObject — actor fuer thread-safety)
- `private var tasks: [UUID: Task<Void, Never>]` — eine SSE-Task pro Roboter
- `private var isConnected: [UUID: Bool]` — SSE-Status pro Roboter

Methoden:
- `func isSSEActive(for robotId: UUID) -> Bool` — liest `isConnected[robotId]`
- `func connect(robotId: UUID, api: ValetudoAPI, onAttributesUpdate: @escaping @Sendable ([RobotAttribute]) -> Void, onConnectionChange: @escaping @Sendable (Bool) -> Void)` — cancelled bestehende Task fuer robotId, erstellt neue Task die `streamWithReconnect()` ausfuehrt
- `private func streamWithReconnect(...)` — while-loop bis Task.isCancelled: api.streamStateLines() aufrufen, `isConnected[robotId] = true` setzen, `onConnectionChange(true)` callen, dann for-await-in `bytes.lines` iterieren. Jede Zeile die mit `data:` beginnt: JSON nach `data:` extrahieren, als `[RobotAttribute]` decoden, `onAttributesUpdate` aufrufen. Bei CancellationError: break (NICHT swallowing!). Bei anderem Error: `isConnected = false`, `onConnectionChange(false)`, 30s sleep, retry (per D-03)
- `func disconnect(robotId: UUID)` — task cancellen, aus beiden Dictionaries entfernen
- `func disconnectAll()` — alle Tasks canceln, beide Dictionaries leeren

**RobotManager um SSE-Integration erweitern (per D-04, D-05, D-06):**

1. Neue Property: `private let sseManager = SSEConnectionManager()` am Anfang der Klasse

2. `startRefreshing()` komplett umbauen (per D-04 — nie SSE und Polling gleichzeitig):
   - refreshTask = Task mit while-loop (wie bisher)
   - Am Anfang jeder Iteration: fuer jeden Robot SSE-Connect versuchen falls noch nicht verbunden
   - `await sseManager.connect(robotId:api:onAttributesUpdate:onConnectionChange:)` aufrufen
   - `onAttributesUpdate` Callback: `applyAttributeUpdate(attrs, for: id)` auf MainActor aufrufen (neue private Methode, siehe unten)
   - `onConnectionChange` Callback: `sseConnectionChanged(connected, for: id)` auf MainActor aufrufen
   - Polling nur fuer Robots OHNE aktive SSE: `let sseActive = await sseManager.isSSEActive(for: robot.id)` pruefen vor `refreshRobot()`
   - 5s sleep am Ende beibehalten (fuer Polling-Fallback und SSE-Reconnect-Check)

3. Neue private Methode `applyAttributeUpdate(_ attrs: [RobotAttribute], for id: UUID)`:
   - Aktuellen robotStates[id] lesen
   - Neuen RobotStatus mit `isOnline: true`, neuen attributes, bestehender info erstellen
   - `checkStateChanges()` fuer Notifications aufrufen (per bestehendem Pattern)
   - `previousStates[id]` und `robotStates[id]` aktualisieren (per D-05 — bestehende View-Bindings bleiben)

4. Neue private Methode `sseConnectionChanged(_ connected: Bool, for id: UUID)`:
   - Wenn `connected == false`: Logger-Warnung loggen
   - ErrorRouter-Integration per D-06: `ErrorRouter.shared.show()` mit reconnecting-Nachricht wenn SSE dropped
   - Wenn `connected == true`: Logger-Info loggen (SSE wiederhergestellt)

5. `removeRobot(_ id:)` erweitern: `Task { await sseManager.disconnect(robotId: id) }` aufrufen BEVOR apis/states gecleared werden

6. Redundante `checkConnection()` in `refreshRobot()` entfernen: Stattdessen direkt `getAttributes()` + `getRobotInfo()` aufrufen und Fehler als offline-Signal werten (Research CONCERNS.md Punkt). Das ist ein bestehender Performance-Overhead — spart einen HTTP-Request pro Poll-Zyklus.
