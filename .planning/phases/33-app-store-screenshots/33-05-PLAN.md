  <action>
    Validierungs-Pass über alle 18 PNGs. Resultate in VALIDATION.md festhalten.

    **Validierungs-Checks:**

    1. **Dimension exakt 1320x2868:**
       ```bash
       for f in ValetudoApp/AppStore/screenshots/export/de/*.png ValetudoApp/AppStore/screenshots/export/en/*.png; do
         w=$(sips -g pixelWidth "$f" | awk '/pixelWidth/ {print $2}')
         h=$(sips -g pixelHeight "$f" | awk '/pixelHeight/ {print $2}')
         if [ "$w" = "1320" ] && [ "$h" = "2868" ]; then
           echo "OK $f: ${w}x${h}"
         else
           echo "FAIL $f: ${w}x${h}"
         fi
       done
       ```

    2. **PNG-Format:**
       ```bash
       for f in ValetudoApp/AppStore/screenshots/export/de/*.png ValetudoApp/AppStore/screenshots/export/en/*.png; do
         file "$f"
       done
       ```
       Erwartung: alle "PNG image data, 1320 x 2868"

    3. **Farbraum (sRGB empfohlen):**
       ```bash
       for f in ValetudoApp/AppStore/screenshots/export/de/*.png ValetudoApp/AppStore/screenshots/export/en/*.png; do
         sips -g space -g profile "$f" | tr '\n' ' '
         echo
       done
       ```
       Wenn Farbraum != sRGB, konvertieren:
       ```bash
       sips -m "/System/Library/ColorSync/Profiles/sRGB Profile.icc" "$f" --out "$f"
       ```

    4. **Alpha-Kanal-Check** (Apple akzeptiert beides, aber konsistent ist sauberer):
       ```bash
       for f in ValetudoApp/AppStore/screenshots/export/de/*.png ValetudoApp/AppStore/screenshots/export/en/*.png; do
         sips -g hasAlpha "$f"
       done
       ```

    5. **Filesize-Sanity-Check:**
       ```bash
       du -k ValetudoApp/AppStore/screenshots/export/de/*.png ValetudoApp/AppStore/screenshots/export/en/*.png | sort -n
       ```
       Erwartung: 500KB - 8MB pro PNG. Wenn deutlich groesser (>10MB) → mit `pngquant --quality=90-100 --output {file} {file}` komprimieren.

    6. **Filename-Konsistenz:**
       ```bash
       diff <(ls ValetudoApp/AppStore/screenshots/export/de/) <(ls ValetudoApp/AppStore/screenshots/export/en/)
       ```
       Erwartung: kein Output (identische Dateinamen in beiden Ordnern).

    | de/01-live-map.png | 2.3 MB | OK |

    ## Filename-Konsistenz
    DE-Ordner und EN-Ordner: identische Filenames ✓

    |------|-------|--------|-------------|-------------|
    | 1 | 01-live-map.png | Live-Karte | Live dabei. | Watch live. |

    | 6 | 06-update-flow.png | Update-Flow | Firmware frisch, ohne Cloud. | Fresh firmware, no cloud. |
    | 7 | 07-robot-list.png | Roboter-Übersicht | Mehrere Roboter? Kein Problem. | More than one? No problem. |

    | 9 | 09-valetudo-control.png | Volle Valetudo-Kontrolle | Alles was Valetudo kann. | Everything Valetudo does. |

    ## Upload-Workflow

    ### Schritt 1 — App Store Connect öffnen
    1. Login auf https://appstoreconnect.apple.com/
    2. Apps → ValetudiOS auswählen
    3. App Store Tab → unter iOS App die aktuelle Version auswählen (oder "Neue Version vorbereiten")

       05-notifications.png, 06-update-flow.png, 07-robot-list.png, 08-consumables.png,
       09-valetudo-control.png
    3. Apple validiert automatisch (Größe, Format) — bei Reject Fehlermeldung lesen, ggf. PNG erneut prüfen

    2. Section Screenshots → iPhone 6.9" Display
    3. Gleiche 9 Files aus `export/en/` in identischer Reihenfolge hochladen

    | Reject | Fix |
    |--------|-----|
    | "Wrong dimensions" | Sips-Check: muss 1320x2868 sein. Re-Export mit pixelRatio: 1 |
    | "Misleading content" | Headlines müssen funktional korrekt sein. Alle aktuellen sind faktisch (z.B. "Live dabei" = Live-Karte tatsächlich in App) |

    | "Different language than locale" | App-Sprache im Source-Shot muss zur Locale passen — DE-Locale = DE-App-Texte. Aus Plan 03 validiert |

    FR-Screenshots sind explizit deferred (siehe CONTEXT.md). Apple braucht keine FR-Screenshots — FR-User sehen die EN-Screenshots als Fallback. Nachreichbar in einer späteren Phase.
